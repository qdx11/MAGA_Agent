⚙ Tool Registry (동적 관리)

v3: 하드코딩

AVAILABLE_TOOLS = {
    "excel": ["ExcelStructureParser", ...],
    "mes": ["MESQueryTool", ...],
}

PREREQUISITE_RULES = {
    "HeaderDetector": ["ExcelStructureParser"],
    ...
}
⚠ 새 Tool 추가 시 코드 직접 수정 필요

v4: YAML + Auto-generate

tools:
  - name: OutlierDetector
    group: excel
    prerequisites:
      - ExcelStructureParser
      - HeaderDetector
    retry_strategies:
      - adjust_parameters
      - aggregate_only
    entry_point: "tools.excel..."
✓ YAML 등록만으로 Planner/Executor 자동 반영

↻ Retry 전략 (툴별 개별 지정)

Excel 분석 Tools

  • 1 adjust_parameters — 헤더행/시트 재지정
  • 2 try_vision_fallback — 이미지 변환 후 Vision
  • 3 aggregate_only — 원본 포기, 집계만

MES 조회 Tools

  • 1 adjust_parameters — 날짜/라인 범위 조정
  • 2 reduce_scope — 기간 축소 후 재조회

RAG Tools

  • 1 adjust_parameters — 쿼리 재작성
  • 2 expand_search — 검색 범위 확장
* 각 Tool의 retry_strategies는 config/tools.yaml에서 관리. 코드 수정 없음.

⇄ v3 → v4 비교

항목 v3 v4
Tool 목록 코드에 하드코딩 config/tools.yaml + 동적 Registry
선행조건 규칙 코드에 하드코딩 Registry에서 자동 생성
검증 시점 전체 실행 후 Critic만 매 스텝 후 Step Guard + 최종 Critic
중간 실패 전체 재실행 Replanner가 남은 플랜만 수정
Retry 전략 전역 하드코딩 (1→2→3) 툴별 retry_strategies in YAML
Executor LLM retry 시 LLM 호출 retry에도 코드 레벨 (구조화 JSON)
State 크기 무한 누적 sliding window + 요약
관측성 없음 Tracer (노드별 로깅/타이밍)
멀티유저 없음 UserContext + 권한 체크
LLM 장애 전체 중단 CircuitBreaker + fallback LLM
테스트 없음 MockLLM + 노드 단위 테스트
MCP 전환 미고려 entry_point → MCP endpoint 교체 가능
